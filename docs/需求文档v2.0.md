# 知识库管理系统 - 需求文档 v2.0

## 一、项目概述

### 1.1 项目背景
- **项目名称**：知识库助手（Knowledge Base Assistant）
- **项目类型**：桌面应用程序（仅 Windows PC 端）
- **核心功能**：整合多来源文档，建立个人知识库，支持 AI 辅助知识管理
- **目标用户**：需要整理和管理知识的专业人士（以文本工作者、知识管理者为主）

### 1.2 核心设计原则
1. **本地优先**：所有数据存储在本地，不依赖云服务
2. **只读同步**：对 Obsidian 源文件只读取、不修改
3. **安全第一**：不改变用户原有工作文档
4. **AI 辅助**：利用 AI 提升知识整理效率

## 二、功能需求

### 2.1 知识来源管理

#### 2.1.1 Obsidian 笔记同步
| 功能 | 描述 |
|------|------|
| 指定 Obsidian vault 路径 | 用户选择 Obsidian 仓库文件夹 |
| 自动分类读取 | 读取 Obsidian 已有分类结构 |
| 只读模式 | 同步后不修改 Obsidian 任何文件 |
| 增量同步 | 只同步新增/修改的笔记 |
| 手动/定时同步 | 支持手动触发或定时自动同步 |
| 排除规则 | 支持排除特定文件夹/文件 |

#### 2.1.2 文档导入（支持格式）
| 格式 | 处理方式 |
|------|----------|
| Markdown (.md) | 直接导入，支持 frontmatter |
| Word (.docx) | 提取文本，保留基本格式 |
| Excel (.xlsx) | 提取表格数据 |
| PDF | 提取文本内容 |
| 图片 (.jpg/.png) | OCR 识别文字（可选功能） |
| 音频/视频 | 提取元信息（可选功能） |

### 2.2 知识库存储

#### 2.2.1 存储结构
```
KnowledgeBase/
├── 文档库/              # 导入的各类文档
│   ├── 技术文档/
│   ├── 工作日志/
│   └── 其他/
├── Obsidian同步/        # 从 Obsidian 同步的内容
│   ├── 按原始分类/
│   └── 已整理/
├── 模板/                # 笔记模板
└── 附件/                # 导入的媒体文件
```

#### 2.2.2 元数据管理
- 标题、标签、分类
- 创建时间、修改时间
- 来源类型（Obsidian/导入/手动创建）
- 引用关系

### 2.3 知识检索与分析

#### 2.3.1 搜索功能
- 全文搜索
- 按标题/标签/分类搜索
- 搜索结果高亮

#### 2.3.2 统计分析
- 知识总量统计（按类型、分类）
- 文档数量趋势
- 标签热度分析
- 来源分布统计

### 2.4 知识库操作

#### 2.4.1 增删改查
| 操作 | 描述 |
|------|------|
| 创建 | 新建笔记、从模板创建 |
| 读取 | 查看笔记内容 |
| 更新 | 编辑笔记内容、修改元数据 |
| 删除 | 移动到回收站、可恢复 |

#### 2.4.2 分类管理
- 创建/重命名/删除分类
- 拖拽移动笔记
- 批量分类调整

#### 2.4.3 导入/导出
- 导出格式：Markdown、PDF、HTML
- 批量导出
- 数据备份/恢复

### 2.5 AI 辅助功能

#### 2.5.1 智能标签
- AI 分析内容自动建议标签
- 一键批量打标签

#### 2.5.2 内容摘要
- 自动生成笔记摘要
- 关键信息提取

#### 2.5.3 知识问答
- 基于知识库的 AI 对话
- 参考引用来源

## 三、用户界面需求

### 3.1 布局设计

#### 主窗口布局
```
┌─────────────────────────────────────────────────────┐
│  菜单栏  │  文件  编辑  视图  工具  帮助            │
├─────────┬───────────────────────────┬───────────────┤
│ 侧边栏  │       主内容区            │   AI 面板     │
│         │                           │  （可折叠）   │
│ • 分类  │  • 笔记列表               │               │
│ • 标签  │  • 编辑器                 │  • 问答       │
│ • 统计  │                           │  • 建议       │
└─────────┴───────────────────────────┴───────────────┘
│                    状态栏                         │
└─────────────────────────────────────────────────────┘
```

### 3.2 核心界面

#### 主界面
- 三栏布局：侧边栏 + 主内容区 + AI 面板
- 侧边栏：分类树、标签云、快捷入口
- 主内容区：笔记列表 + Markdown 编辑器
- AI 面板：对话、建议、统计

#### 设置界面
- 知识来源配置（Obsidian 路径、导入目录）
- 同步设置
- 主题设置（浅色/深色）
- AI 配置（API Key）

## 四、技术架构

### 4.1 技术栈
- **框架**：Electron + React + TypeScript
- **样式**：Tailwind CSS
- **状态管理**：Zustand
- **存储**：本地文件系统 + JSON
- **AI**：Anthropic Claude API（用户需自行配置 Key）

### 4.2 数据流
```
用户操作 → React 组件 → Zustand Store → Electron IPC → 主进程 → 文件系统
                                    ↓
                              AI 服务调用
```

## 五、非功能需求

### 5.1 性能需求
- 启动时间 < 3 秒
- 1000 篇笔记内搜索响应 < 500ms

### 5.2 兼容性
- Windows 10/11
- 仅桌面端使用

### 5.3 数据安全
- 不修改源文件
- 本地存储不外传
- 支持数据备份
