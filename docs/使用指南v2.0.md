## 一、快速开始

### 1.1 首次运行

1. **启动应用**
   ```powershell
   cd d:\cc-projects\projects\knowledge-base
   ```

## 三、基础功能

### 3.1 创建笔记

**方法一：手动创建**
1. 在笔记列表上方点击 "+" 按钮
2. 输入笔记标题
3. 选择分类（默认：技术文档）
4. 点击创建

**方法二：从模板创建**
1. 切换到侧边栏「模板」标签
2. 点击模板笔记
3. 自动创建基于模板的新笔记

### 3.2 编辑笔记

1. 点击笔记列表中的笔记
2. 在主编辑区域修改内容
3. 按 `Ctrl+S` 或点击「保存」按钮保存

### 3.3 删除笔记

1. 鼠标悬停在笔记上
2. 点击右侧出现的删除图标
3. 笔记会移到「回收站」

### 3.4 搜索笔记

1. 在顶部搜索框输入关键词
2. 支持按标题、内容、标签搜索
3. 搜索结果实时过滤

### 3.5 主题切换

1. 点击设置按钮
2. 找到「外观」下拉菜单
3. 选择「浅色」或「深色」
4. 保存设置

## 四、知识来源管理

### 4.1 Obsidian 同步（只读）

将 Obsidian 中的笔记同步到知识库，**不修改 Obsidian 任何文件**。

#### 配置步骤

1. **获取 Obsidian  vault 路径**
   - 打开 Obsidian
   - 右键点击左侧仓库名称
   - 选择「显示详情」
   - 复制「库位置」路径

2. **在知识库助手中配置**
   - 点击设置按钮
   - 找到「Obsidian 同步」部分
   - 点击「选择」按钮
   - 粘贴刚才复制的路径

3. **验证配置**
   - 系统会自动验证路径是否有效
   - 有效会显示「✓ 有效的 Obsidian vault」

4. **开启自动同步**（可选）
   - 开启后，每次启动应用会自动同步
   - 关闭则需要手动同步

#### 手动同步

同步按钮位置：在设置界面的 Obsidian 配置中（或者后续版本会放在主界面）

同步会：
- 读取 Obsidian vault 中的 .md 文件
- 复制到知识库的「Obsidian同步」文件夹
- 保留原始分类结构
- 添加来源标记（不会被同步回去）

#### 同步说明
- ✅ 只读同步，不修改 Obsidian 任何文件
- ✅ 增量同步，只同步新增/修改的笔记
- ✅ 可排除特定文件夹

### 4.2 文档导入

支持将 Word、Excel、PDF 等格式导入为 Markdown。

#### 支持格式

| 格式 | 说明 |
|------|------|
| .md | Markdown 文件，直接导入 |
| .txt | 纯文本文件 |
| .docx | Word 文档，提取文字内容 |
| .xlsx | Excel 表格，转换为 Markdown 表格 |
| .pdf | PDF 文档，提取文字 |

#### 导入步骤

1. 点击顶部工具栏「导入」按钮
2. 选择导入到的分类
3. 点击「选择文件导入」
4. 选中一个或多个文件
5. 点击「导入」
6. 等待导入完成

导入后的文件会：
- 转换为 Markdown 格式
- 保存到选择的分类目录下
- 添加来源元数据（来源文件、导入时间）

### 4.3 知识统计

查看知识库的统计分析。

1. 点击顶部工具栏「统计」按钮
2. 查看以下数据：
   - 总笔记数

   - 知识来源分布（Obsidian/导入/手动）
   - 分类分布（条形图）
   - 热门标签（标签云）

## 五、知识库操作

### 5.1 回收站

删除的笔记不会永久删除，而是移到回收站。

**位置**：侧边栏底部，点击「回收站」

**功能**：
- **恢复**：点击「恢复」按钮，笔记恢复到原来位置
- **永久删除**：点击「删除」按钮，彻底删除（不可恢复）
- **清空回收站**：点击「清空回收站」，删除所有回收站内容

### 5.2 导出功能

支持导出笔记为其他格式。

1. 点击顶部工具栏「导出」按钮
2. 选择导出方式：
   - 「导出当前笔记」：导出正在查看的笔记
   - 「批量导出」：选择多篇笔记导出
3. 选择格式：
   - Markdown (.md)
   - 网页 (.html)
   - 纯文本 (.txt)
4. 点击导出
5. 选择保存位置

## 六、AI 辅助功能

### 6.1 配置 AI

1. 点击设置按钮
2. 在「Claude API Key」中粘贴你的 API Key
3. 点击「测试」验证
4. 保存设置

获取 API Key：
- 访问 https://console.anthropic.com/
- 注册/登录
- 创建 API Key
- 复制到应用设置中

### 6.2 AI 对话

在右侧 AI 面板中：
1. 选择「对话」模式
2. 输入问题
3. 发送
4. AI 会根据知识库内容回答

### 6.3 内容摘要

1. 选择一篇笔记
2. 在 AI 面板选择「摘要」
3. 点击「生成摘要」
4. AI 会生成内容摘要

### 6.4 文字润色

1. 选择一篇笔记
2. 在 AI 面板选择「润色」
3. 点击「润色内容」
4. AI 会提供润色后的版本

## 七、Obsidian 联合使用

### 7.1 工作流程

```
┌─────────────────┐     同步      ┌─────────────────┐
│                 │ ─────────►  │                 │
│   Obsidian      │   只读      │  知识库助手     │
│   (日常记录)    │              │  (整理展示)    │
│                 │ ◄─────────  │                 │
└─────────────────┘   手动编辑   └─────────────────┘
```

### 7.2 使用场景

#### 场景一：日常记录在 Obsidian

1. **日常使用 Obsidian**
   - 用 Obsidian 记录每日工作笔记
   - 按自己习惯的分类整理
   - 利用 Obsidian 的双向链接等高级功能

2. **同步到知识库助手**
   - 在知识库助手设置中配置 Obsidian 路径
   - 点击同步（或开启自动同步）
   - 所有笔记同步到知识库

3. **在知识库助手中**
   - 查看统计报表
   - 使用 AI 辅助整理
   - 导出为其他格式

#### 场景二：分工使用

| 应用 | 用途 |
|------|------|
| Obsidian | 日常记录、写作、双向链接 |
| 知识库助手 | 知识整理、AI 辅助、统计分析、导出分享 |

### 7.3 常见问题

**Q1: 同步会覆盖我在 Obsidian 的修改吗？**
> 不会。同步是单向的（Obsidian → 知识库助手），只读取不修改。

**Q2: 我在知识库助手中修改的笔记会同步到 Obsidian 吗？**
> 不会。知识库助手修改的是副本，不会影响原始 Obsidian 文件。

**Q3: 同步有延迟吗？**
> 手动同步是实时的。自动同步在应用启动时执行一次。

**Q4: 删除的笔记在哪里？**
> 删除的笔记在知识库助手的「回收站」中，Obsidian 中的原文件不受影响。

### 7.4 最佳实践

1. **保持 Obsidian 为主要编辑工具**
   - 所有日常记录在 Obsidian 中完成
   - 利用 Obsidian 的强大功能

2. **定期同步**
   - 开启自动同步，每次打开应用自动同步
   - 或手动点击同步按钮

3. **利用知识库助手做整理**
   - 查看统计分析了解知识结构
   - 使用 AI 辅助生成摘要
   - 导出为其他格式分享

## 八、常见问题

### 8.1 技术问题

**Q: 应用无法启动**
> 1. 检查 Node.js 是否安装正确
> 2. 删除 node_modules 后重新安装：`rm -rf node_modules && npm install`
> 3. 查看控制台错误信息

**Q: 导入文档失败**
> 1. 确认文件格式是否支持
> 2. 检查文件是否被其他程序打开
> 3. 尝试重新保存源文件

**Q: AI 功能不可用**
> 1. 确认已配置正确的 API Key
> 2. 检查网络连接
> 3. 确认 API Key 有余额

### 8.2 使用问题

**Q: 如何备份知识库？**
> 1. 找到设置的「笔记存储目录」
> 2. 整个文件夹复制备份
> 3. 或使用导出功能导出为 Markdown

**Q: 如何移动笔记到其他分类？**
> 目前版本需要手动移动文件到目标分类文件夹，然后刷新应用。

**Q: 回收站可以恢复吗？**
> 可以。在回收站中点击「恢复」按钮即可恢复。
