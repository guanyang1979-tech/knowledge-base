# 质检审核员机制

## 概述
每次功能开发完成后，自动进行代码质量和功能完整性检查。

## 质检清单

### 1. 代码质量检查
- [ ] TypeScript 类型完整，无 `any` 类型（除非必要）
- [ ] 无 console.log 调试代码（生产环境）
- [ ] 错误处理完善
- [ ] 代码结构清晰，命名规范

### 2. 功能完整性检查
- [ ] 主进程 IPC 处理器已添加
- [ ] Preload API 已暴露
- [ ] 前端组件正确调用 API
- [ ] 配置正确保存和加载
- [ ] 边界情况已处理

### 3. 用户体验检查
- [ ] 加载状态提示
- [ ] 错误提示友好
- [ ] UI 响应正常

### 4. 测试验证
- [ ] 开发模式可正常启动
- [ ] 功能可正常使用

---

## 质检流程

```
开发完成 → 自检清单 → 报告问题 → 修复 → 通过
```

每次开发完成后，质检审核员将自动执行以上检查清单。
