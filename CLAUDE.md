# 知识库助手 - 开发指南

## 质检审核员机制（已启用）

每次功能开发完成后，**必须自动执行质检审核**，对照 `docs/质检审核员.md` 中的清单逐项检查。

### 质检流程

```
开发完成 → 自动执行质检清单 → 报告问题 → 修复 → 记录到 docs/修改记录.md → 重新打包
```

### 质检清单

1. **代码质量**：TypeScript 类型完整、无 console.log 遗留、错误处理完善
2. **功能完整性**：主进程 IPC 处理器、Preload API、前端组件调用、配置持久化
3. **用户体验**：加载状态、错误提示、UI 响应
4. **测试验证**：重新构建打包，确保 exe 可正常运行

### 文档留痕

每次改动必须追加记录到 `docs/修改记录.md`，包含：
- 改动日期
- 改动背景
- 改动文件清单
- 设计变更详情
- Bug 修复记录
